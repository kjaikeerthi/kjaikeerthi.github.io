# Tech Stack

## Framework & Runtime
- **Application Framework:** Next.js (Static Site Generation / SSG)
- **Language/Runtime:** TypeScript / Node.js
- **Package Manager:** npm or yarn

## Frontend
- **JavaScript Framework:** React (via Next.js)
- **CSS Framework:** Chakra UI Pro (licensed premium component library)
- **UI Components:** Chakra UI Pro components and theme system
- **Styling Approach:** CSS-in-JS via Chakra UI's styling system
- **Content Format:** Markdown/MDX for blog posts and static content

## Static Content Management
- **Blog Posts:** Local markdown/MDX files stored in repository
- **Resume Data:** Structured data files (JSON/TypeScript) or markdown format
- **Tools List:** Static JSON/TypeScript data files
- **Content Updates:** Direct file edits committed to git repository

## Build & Deployment
- **Static Site Generation:** Next.js SSG with static export
- **Hosting:** GitHub Pages with custom domain
- **Custom Domain:** https://jaikeerthi.in (CNAME configured)
- **CI/CD:** GitHub Actions for automated builds and deployments

## Development Tools
- **Linting:** ESLint with TypeScript support
- **Formatting:** Prettier
- **Version Control:** Git / GitHub
- **Type Checking:** TypeScript compiler

## Additional Libraries
- **Syntax Highlighting:** Shiki for code blocks in blog posts
- **SEO:** next-seo package for meta tag management
- **Analytics:** Google Analytics
- **Icons:** Chakra UI icons or React Icons
- **Date Handling:** date-fns or Day.js for date formatting
- **RSS Feed:** next-rss for blog feed generation

## Content & Data Structure
- Static content stored in git repository
- No database or external CMS required
- Version-controlled content management
- Fast build times and optimal performance
- Secure and maintenance-free hosting

## Development Workflow
- Local development with hot reload
- TypeScript for type safety
- Component-driven development with Chakra UI Pro
- Markdown/MDX for content authoring
- Git-based content versioning
